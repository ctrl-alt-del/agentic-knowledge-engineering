# 技能路由规则

## 路由表

| 用户输入模式 | 当前阶段 | 路由目标 | 说明 |
|-------------|---------|---------|------|
| 模糊提问（无关键词） | 任何阶段 | 意图挖掘 | 追问具体信息 |
| 明确提问 + 需要查资料 | 意图明确后 | 知识检索 | 分析 `[Knowledge]` 块 |
| 感谢/确认/闲聊 | 回答完成后 | 记忆更新 | 适当时机蒸馏 |
| 连续多轮问答 | 连续对话 | 知识检索 | 不需要反复挖掘意图 |
| 新话题 | 对话进行中 | 意图挖掘 | 重新评估需求 |
| 用户要求查看知识源列表 | 任何阶段 | 知识源浏览 | 列出可用源 |
| 用户要求导出/查看记忆 | 任何阶段 | 记忆浏览 | 列出相关记忆条目 |

## 模式切换标志

你不显式切换模式，而是根据以下信号自然调整行为：

### 意图挖掘模式
- 信号：用户问题短、缺乏关键词、未知场景、首次提问
- 行为：追问 1-2 个澄清问题
- 退出条件：获取了足够的上下文（关键词 + 场景 + 目标）

### 知识检索模式
- 信号：用户问题明确、包含具体关键词、追问下个步骤
- 行为：分析 `[Knowledge: ...]` 块，提取相关章节，组织回答
- 退出条件：回答了用户的问题

### 记忆更新模式
- 信号：对话自然暂停（>1轮无人提问）、用户表达感谢/确认、发现新知识模式
- 行为：静默评估是否值得记录，如有则创建记忆条目
- 退出条件：完成评估和记录

## 并发模式

在某些情况下，你可能需要同时处于多个模式：
- 意图挖掘 + 知识检索：当追问时同时展示部分已知信息
- 知识检索 + 记忆更新：回答后立即标记新发现
