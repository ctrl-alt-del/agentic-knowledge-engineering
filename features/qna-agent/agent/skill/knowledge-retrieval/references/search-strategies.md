# 搜索策略

## 通用搜索流程

1. 接收用户查询
2. 从查询中提取关键词
3. 在倒排索引中查找匹配的 token-文件对
4. 按 TF 分数聚合排序
5. 选择 top-K（默认5）个知识块
6. 获取知识块完整内容（浏览器缓存）
7. 注入为 `[Knowledge: source#section]` 上下文块
8. 发送给 LLM 推理

## 按知识源类型优化

### 正向文档（forward）
- 关键词提取：文档标题词 + 章节标题词 + 正文关键词
- 偏好：精确匹配 > 模糊匹配
- 示例：用户问「如何部署」→ 优先匹配 `deploy` 和 `部署` 词条

### 代码知识（reverse）
- 关键词提取：错误名 + 类名 + 函数名 + 模块路径
- 偏好：错误名精确匹配 > 模糊匹配
- 示例：用户问「NullPointerException 在哪里」→ 优先匹配 `nullpointerexception`

### 胶囊知识（capsule）
- 关键词提取：时间 + 事件名 + 频率词 + 因果词
- 偏好：因果链匹配 > 单点匹配
- 示例：用户问「超时导致失败怎么做」→ 匹配 `timeout` + `failed` + `cause`

## 中英文混合搜索

- 中文文本：按字符 bigram 分词（「部署失败」→「部署」「失败」）
- 英文文本：按空格分词 + 小写化
- 混合文本：先按语言切分，再分别分词
- CJK 字符单独提取，英文单词整体匹配

## 排名信号权重

| 信号 | 权重 | 说明 |
|------|------|------|
| 标题匹配 | 3x | 章节标题包含查询词 |
| TF 分数 | 1x | 词频在章节中的密度 |
| 知识源类型 | 1.5x | 匹配用户意图的类型 |
| 新鲜度 | 0.5x | 索引构建时间越近越好 |
| 历史点击 | 1x | 过去的交互中该章节被引用次数（记忆系统中追踪） |
