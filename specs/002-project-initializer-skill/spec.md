# 项目初始化助手技能 — 规范

## 用户故事
作为用户，我希望通过自然对话完成知识工程平台的初始化配置，而不需要在复杂表单中手动填写每一项。

## 对话流程

1. **恢复检查**：检查是否存在未完成的表单状态，若有则询问是否继续
2. **基础信息**：收集项目名称（≤20字符）和项目描述（≤50字符）
3. **能力选择**：介绍四种知识处理能力模块，根据对话意图推荐，用户确认
4. **模板选择**：提供三个预设模板，用户可单选或自定义
5. **子配置**：对每个启用的能力深入询问子配置项
6. **确认输出**：汇总已配置项，用户确认后输出结构化 JSON

## 验收标准

### 正常流程
- [ ] 给定用户开始对话，当表达项目意图后，则 LLM 按阶段引导收集信息
- [ ] 给定用户是新手，当询问推荐时，则 LLM 根据描述推荐合适的模板和能力组合
- [ ] 给定所有字段填写完成，当用户确认后，则输出符合 schema 的 JSON
- [ ] 给定对话中断后恢复，当检测到未完成状态时，则从中断处继续填写
- [ ] 给定用户修改某项后，当确认时，则更新对应字段并重新显示汇总

### 边界情况
- [ ] 给定项目名称超过20字符，当输入时，则提示长度限制并要求缩短
- [ ] 给定描述超过50字符，当输入时，则提示限制
- [ ] 给定未选择任何能力，当尝试进入下一步时，则提示至少选一项
- [ ] 给定仓库 URL 格式错误，当输入时，则提示格式要求并提供示例
- [ ] 给定模板选择后，当用户想调整某项时，则允许覆盖模板预设值

## 非功能需求
- 所有对话使用中文
- 输出 JSON 可被程序直接解析
- MCP/API 端点通过配置文件注入，不硬编码在技能中
- 支持流式输出以便 Chat UI 逐字显示
